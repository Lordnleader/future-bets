# Thread Rules

UI refinement threads should not change pipeline logic.

Data pipeline threads should not redesign the UI.

Schema threads should focus on data shapes and validation.

Deployment threads should focus on build, hosting, and workflow concerns.

Bug threads should fix one issue at a time.

Each new thread should begin by inspecting the current codebase and summarising:

- what already exists
- what is incomplete
- the smallest change needed

Only then should implementation begin.
